Github Pages 设置

  • 首先需要创建一个 Github 个人账户
  • 每个帐号只能有一个仓库来存放个人主页，而且仓库的名字必须是 username/username.github.io，这是特殊的命名约定
  • 个人主页的网站内容是在master分支下
  • 需要有一次提交，然后就可以通过 http://username.github.io 来访问你的个人主页

部署 Hexo 到 Github Pages

原理

  • Hexo -g 会生成一个静态网站（看起来就是一个 public 文件夹）
  • 只要将这个文件夹下的所有内容提交到 Github 刚刚创建的那个仓库下就能被访问到了

部署方式一：直接同步文件夹

将 Github 主页仓库同步到本地，直接将 public 文件夹下的内容复制到仓库进行同步

Github 域名绑定

首先需要买一个域名并配置DNS解析，类型为 CNAME，我的值为 wanglejiang.github.io

在 Github 主页仓库创建一个名为 CNAME 的文件，编辑内容为配置的域名即可，但这种方式可能会被本地提交文件的时候覆盖，所以推荐以下方法：

把 CNAME 文件放到本地 blog/source 下，这样可以在打包的时候把 CNAME 打包到 public 文件夹

然后输入域名就可以访问到博客了
